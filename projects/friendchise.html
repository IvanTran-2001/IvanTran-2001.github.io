FriendChise

A task management and knowledge-sharing platform for franchise businesses that connects franchisees to community-driven best practices — built to create consistency and prevent costly mistakes from day one.

Full-Stack Multi-Tenant RBAC SaaS V2 — Live In Pilot
Live Demo View Code Full Case Study

The Problem

New franchisees struggle when opening a store — they lack operational knowledge that established stores have already figured out through trial and error. This hurts the new store and the overall brand. Consistency breaks down. Quality varies. Knowledge stays trapped inside the heads of a few people at a few locations.

I worked at a donut franchise that started as a top 5 location out of 30 and dropped to bottom 2 within two years. Workers were thrown in without training. 12-hour shifts with unfair task distribution led to burnout. Managers blamed workers for problems caused by a lack of structure.

The problems weren't the people — it was the system. FriendChise is the tool I wish existed.

The Solution

A platform that captures operational wisdom from successful stores and makes it accessible to everyone — while automating the boring stuff (scheduling, task distribution) so managers can focus on quality. Imagine if every franchisee performed as well as your best franchisee. That's the goal.

Franchise owners build a task library and define what a standard week looks like. Managers apply that template to any week in one click and track completion through the day. Workers see their schedule on any device and leave tips for each other in each task's comment thread — so knowledge stops being lost and starts compounding across the network.

What's Live

Auth & Access

  • Google OAuth & LinkedIn Sign-In — Auth.js v5, JWT sessions, httpOnly cookies, Edge middleware route protection
  • Demo Mode — isolated sandbox, no account required, auto-expires after 2 hours
  • Role & Permission System — custom roles with 6 permission types; RBAC guards on every server action and API route
  • Production / Demo Isolation — the Walker's pilot and public demo run on completely separate Supabase instances with separate OAuth clients

Task Management & Scheduling

  • Task Library — CRUD with markdown descriptions, color, image upload, role eligibility, tags; card & list views; sort and filter; any org can publish tasks to the franchise network — knowledge flows in both directions
  • Franchise Task Discovery — browse tasks published by any org in the network; selectively add them to your own library; inheritance is opt-in, not a forced sync
  • Task Comments & Voting — threaded comments with upvotes, downvotes, replies, pinning, and soft delete; visible across the franchise for network-scoped tasks
  • Timetable — weekly/daily calendar and simple list view; status tracking (TODO → IN_PROGRESS → DONE); role filter; per-entry edit and reassign
  • Timetable Templates — drag-and-drop weekly schedule builder (calendar or table mode); apply to any date range with frozen task snapshots; duplicate, rename, delete

Operational Tools

  • Dashboard — today's schedule progress bar, member/task/role/completion stats, recent tool activity
  • Conversion Tool — scale ingredient quantities across batch sizes; shareable named conversion sets with full rate lists
  • Staff Roster — multi-week shift scheduling grid; templates with cycle-week repeat; per-day headcount requirements; open-slot bot members
  • Item List — searchable ingredient and product catalog shared across the org

Organization & People

  • Member Management — invite by email, accept/decline flow, role assignment; card & list views with search and role filter; edit, restrict, and remove members; bot slots (placeholder accounts for roster)
  • Notification Panel — bell icon with unread badge; member and franchise invite cards with accept/decline; seen/unseen tracking; Popover on desktop, bottom Sheet on mobile
  • Org Settings — logo upload (512×512 crop dialog), org name, address, timezone, operating days and hours; ownership transfer; org delete
  • Franchise Management — generate invite tokens, view and manage child franchisees, extend/revoke tokens, change franchisee owner; new franchisees inherit roles and settings on join, then selectively adopt tasks from the shared network

What This Demonstrates

Product Thinking

Started from a real problem I lived through. 7 personas, 34 user stories, 36 MoSCoW-prioritized features, and 9 user flows defined before writing code.

System Design

Hierarchical multi-tenant architecture (Brand → Franchisee). Two-layer timetable system. Min/max frequency constraints. Documented tradeoffs. ActionSidebar pattern: a single component that renders as an inline panel on desktop and slides up as a bottom Sheet on mobile — no separate popup needed, consistent behavior everywhere.

Full-Stack TypeScript

Next.js 16 App Router with RSC, Server Actions, and REST API. Discriminated union result types for clean error handling. End-to-end type safety with Zod.

Auth & Security

OAuth + JWT sessions, Edge runtime compatibility, org-scoped permission checks on every route. Full RBAC — not just “admin vs user.”

Production Engineering

582 automated tests: 468 unit (Vitest), 93 integration against a real Postgres DB, 21 Playwright E2E. GitHub Actions CI runs type-check → lint → unit → integration on every push. Sentry error monitoring + session replay. Upstash Redis sliding-window rate limiting on every API route.
