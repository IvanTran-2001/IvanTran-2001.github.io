Project Overview

This technical assessment involved building a Hacker News Article Sort Validator using Node.js and Microsoft's Playwright framework. The task required scraping the 100 newest articles from Hacker News and validating their chronological ordering, demonstrating proficiency in test automation, data validation, and quality assurance engineering.

  • Fetch exactly the first 100 articles from Hacker News/newest using Playwright.
  • Validate articles are sorted newest to oldest by timestamp.
  • Report any ordering inconsistencies with detailed error information.
  • Handle edge cases: invalid timestamps, pagination, and error pages.
  • Provide configurable output options (article count, titles to display, verbose mode).
  • Implement robust error handling and logging throughout.

Technology Stack

Node.js

JavaScript runtime environment for server-side execution

Playwright

Browser automation framework for scraping and testing

JavaScript ES6+

Modern JS with async/await, destructuring, and modules

@playwright/test

Built-in testing framework for unit and integration tests

Key Features

Playwright Scraping

Multi-page navigation across Hacker News with dynamic content extraction and browser automation.

Sort Validation

Parses ISO timestamps and detects any out-of-order articles with precise index and comparison data.

CLI Interface

User-friendly command-line tool with optional flags, default values, and built-in help documentation.

Modular Architecture

Code organised into focused, reusable modules: CLI parser, validator, logger, result handler, and debug utilities.

Error Handling

Gracefully handles pagination failures, invalid timestamps, network issues, and error pages.

Error Injection

Intentional error injection mode to validate the sorting algorithm itself during development.

Module Structure

  • index.js — Main orchestrator coordinating scraping, validation, and output.
  • cli.js — Parses command-line arguments and provides help documentation.
  • validator.js — Core logic for timestamp parsing and sort-error detection.
  • result.js — Result class encapsulating article data and analysis methods.
  • logger.js — Singleton logger for collecting and reporting warnings.
  • debug.js — Test error injection utilities for validating the validator.

Usage Examples

# Basic usage — fetch and validate 100 articles
node index.js

# Show first 5 article titles with validation
node index.js 100 5

# Verbose mode with detailed error reporting
node index.js 100 5 --verbose

# Test mode with intentional sorting errors
node index.js 100 0 --test-error

# Display help
node index.js --help

Testing

All tests run using Playwright's test framework: npx playwright test

  • CLI Parser Tests: Validates argument parsing, defaults, and flag handling.
  • Validator Tests: Timestamp parsing and error detection across edge cases.
  • Result Tests: Verifies Result class methods and error reporting.
  • Integration Tests: End-to-end validation of the full workflow.
  • Test-Error Tests: Confirms error injection behaves correctly.
