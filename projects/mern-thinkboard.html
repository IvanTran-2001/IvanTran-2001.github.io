📝 MERN Thinkboard

Full-Stack Authenticated Note-Taking Application

🎯 Project Overview

MERN Thinkboard is a production-ready, full-stack note-taking application built with the MERN stack (MongoDB, Express, React, Node.js). The application features secure JWT-based authentication, protected routes on both frontend and backend, and a modern, responsive UI powered by TailwindCSS and DaisyUI. Users can securely create, manage, search, and organize their personal notes with real-time updates and smart rate limiting.

  • MVC Pattern: Clear separation between Controllers, Routes, Models, and Middleware

  • Route Guards: Frontend GuestRoute and ProtectedRoute components

📚 Key Implementation Details

Authentication Flow

Users register or login through the authentication endpoints. Upon successful authentication, the server generates a JWT token (7-day expiration) and sends it back to the client. The client stores this token and includes it in all subsequent requests via Axios interceptors. The backend middleware validates the token before allowing access to protected routes.

Data Privacy

Each user's notes are isolated from other users through user-specific database queries. When a note is created, updated, or deleted, the backend verifies that the requesting user owns that note before performing the operation, preventing unauthorized data access.

Rate Limiting Strategy

IP-based rate limiting protects authentication endpoints from brute-force attacks. User-based rate limiting protects general API endpoints from abuse. This dual-layer approach ensures both security and fair resource usage while leveraging Upstash Redis for serverless scalability.

📁 Project Structure

The project is organized with clear separation of concerns:

  • frontend/ - React application with components, pages, and utilities
  • backend/src/ - Express server with controllers, routes, models, and middleware
  • config/ - Database and Redis configuration
  • routes/ - Auth and notes API routes
  • controllers/ - Business logic for authentication and note management
  • models/ - Mongoose schemas for Users and Notes
  • middleware/ - Authentication and rate limiting middleware
