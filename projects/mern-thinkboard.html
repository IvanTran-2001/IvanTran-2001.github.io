Project Overview

MERN Thinkboard is a full-stack, production-ready note-taking application showcasing modern web development practices. Built with the MERN stack (MongoDB, Express, React, Node.js), it demonstrates expertise in:

  • Backend Architecture: RESTful API with Express, JWT authentication, middleware pipeline
  • Database Design: MongoDB with optimised queries and strategic indexing
  • Frontend Development: React hooks, state management, protected routes
  • Security: JWT tokens, bcrypt hashing, rate limiting, input validation
  • DevOps: Cloud deployment with MongoDB Atlas and Render

Architecture & Patterns

The application follows industry best practices and modern architectural patterns:

  • MVC Pattern: Clean separation between Controllers, Routes, Models, and Middleware for maintainability
  • RESTful API Design: Semantic HTTP methods (GET, POST, PUT, DELETE) with proper status codes
  • Middleware Pipeline: Authentication and rate limiting middleware for protected routes
  • Axios Interceptors: Automatic JWT token injection in request headers without code duplication
  • Route Guards: Frontend GuestRoute and ProtectedRoute components for route access control
  • Schema Validation: Mongoose schemas enforce data integrity at database level
  • Error Handling: Centralised error handling with meaningful, user-friendly error messages
  • Environment Configuration: Secure environment variable management for production deployment
Security Features
  • JWT Authentication: Stateless authentication with 7-day token expiration for enhanced security
  • Password Hashing: bcrypt with configurable salt rounds for secure password storage
  • Protected Routes: Both frontend and backend protected from unauthorised access
  • Rate Limiting: Prevents brute-force attacks on authentication endpoints using Upstash Redis
  • Input Validation: Email, password, and username format validation on client and server
  • CORS Configuration: Prevents unauthorised cross-origin requests from external domains
  • Environment Variables: Sensitive data like API keys and database URIs never exposed in version control
Performance Optimisations
  • MongoDB Indexing: Database queries optimised with strategic indexing on frequently queried fields
  • Vite Build Optimisation: Lightning-fast development and production builds with code splitting
  • Component Lazy Loading: React components loaded on demand to reduce initial bundle size
  • API Response Caching: Redis integration for caching frequently accessed data
  • Efficient State Management: React hooks for minimal re-renders and optimal performance
Implementation Highlights

Authentication Flow

Users register or login through dedicated authentication endpoints. Upon successful authentication, the server generates a JWT token with a 7-day expiration and sends it back to the client. The client stores this token and includes it in all subsequent requests via Axios interceptors. The backend middleware validates the token before allowing access to protected routes.

Data Privacy & Isolation

Each user's notes are completely isolated from other users through user-specific database queries. When a note is created, updated, or deleted, the backend verifies that the requesting user owns that note before performing the operation. This prevents unauthorised data access and ensures complete data privacy.

Rate Limiting Strategy

A dual-layer rate limiting approach protects both authentication and general API endpoints. IP-based rate limiting prevents brute-force attacks on login/register endpoints, while user-based rate limiting protects general API endpoints from abuse — all via Upstash Redis, eliminating the need to manage Redis infrastructure directly.
